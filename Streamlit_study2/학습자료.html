1. session_state

1-1. 문제 — 카운터가 안 되는 이유

아래 코드를 실행하고 버튼을 5번 눌러보세요.

import streamlit as st

count = 0
if st.button("클릭"):
    count += 1
st.write("클릭 수:", count)
항상 1만 표시됩니다. 버튼을 누르면 rerun(전체 재실행)이 일어나고, count = 0부터 다시 시작하기 때문입니다.

해결: rerun이 일어나도 값이 유지되는 st.session_state에 저장합니다.

1-2. 기본 패턴 — 명시 읽기·쓰기

import streamlit as st

# ① 초기화 가드: 키가 없을 때만 초기값 세팅
if 'count' not in st.session_state:
    st.session_state['count'] = 0

# ② 읽기
st.write("클릭 수:", st.session_state['count'])

# ③ 쓰기
if st.button("클릭"):
    st.session_state['count'] += 1
초기화 가드가 없으면 매 rerun마다 0으로 리셋됩니다.

1-3. 초기화 버튼 추가

import streamlit as st

if 'count' not in st.session_state:
    st.session_state['count'] = 0

col1, col2 = st.columns([3, 1])
with col1:
    if st.button("클릭"):
        st.session_state['count'] += 1
with col2:
    if st.button("초기화"):
        st.session_state['count'] = 0

st.metric("클릭 수", st.session_state['count'])

1-4. 실용 패턴 A — 좋아요 버튼

import streamlit as st

if 'liked' not in st.session_state:
    st.session_state['liked'] = False

label = "❤️ 좋아요 취소" if st.session_state['liked'] else "🤍 좋아요"
if st.button(label):
    st.session_state['liked'] = not st.session_state['liked']

if st.session_state['liked']:
    st.success("좋아요를 눌렀습니다!")

1-5. 실용 패턴 B — key= 자동 연결

위젯에 key=를 지정하면 session_state에 자동으로 저장됩니다.

import streamlit as st

# key='age_range' → st.session_state['age_range']에 자동 저장
age_range = st.slider("나이 범위", 0, 80, (0, 80), key='age_range')

st.write("session_state 값:", st.session_state['age_range'])

1-6. 실용 패턴 C — 사이드바 필터 초기화 버튼

import streamlit as st
import pandas as pd

@st.cache_data
def load_titanic():
    url = 'https://raw.githubusercontent.com/datasciencedojo/datasets/master/titanic.csv'
    return pd.read_csv(url)

df = load_titanic().copy()
ALL_PCLASS = [1, 2, 3]

if st.sidebar.button("필터 초기화"):
    st.session_state['pclass'] = ALL_PCLASS
    st.session_state['survived_only'] = False

with st.sidebar:
    pclass = st.multiselect(
        "객실 등급", ALL_PCLASS,
        default=st.session_state.get('pclass', ALL_PCLASS),
        key='pclass'
    )
    survived_only = st.checkbox(
        "생존자만",
        value=st.session_state.get('survived_only', False),
        key='survived_only'
    )

filtered = df[df['Pclass'].isin(pclass)]
if survived_only:
    filtered = filtered[filtered['Survived'] == 1]

st.write(f"선택된 승객: {len(filtered):,}명")
st.dataframe(filtered.head(20))
st.session_state.get('key', 기본값) — 키가 없으면 기본값 반환

흔한 오류

오류 원인 해결
KeyError: 'count' 초기화 가드 없이 읽기 if 'count' not in st.session_state: 추가
버튼 눌러도 값 안 바뀜 if st.button(): 밖에서 수정 버튼 블록 안에서 수정
key= + 가드 동시 사용 두 패턴 혼용 패턴 하나만 선택

3. st.form

언제 쓰는가

여러 위젯을 묶어 제출 버튼 한 번에만 집계·차트를 실행하고 싶을 때.

import streamlit as st

with st.form("filter_form"):
    campaign_type = st.selectbox("캠페인 유형", ["Email", "Influencer", "Display"])
    min_roi       = st.slider("최소 ROI", 0.0, 10.0, 0.0)
    submitted     = st.form_submit_button("검색 적용")

if submitted:
    st.write(f"{campaign_type} / ROI >= {min_roi}")
규칙
  • with st.form("고유키"): 블록 안에 위젯 배치
  • st.form_submit_button()이 눌릴 때만 rerun
  • form 안에 st.button() 사용 불가

4. st.file_uploader

import streamlit as st
import pandas as pd

uploaded = st.file_uploader("CSV 파일을 선택하세요", type=["csv"])

if uploaded is not None:
    df = pd.read_csv(uploaded)
    st.success(f"로드 완료: {uploaded.name}  ({len(df):,}행)")
    st.dataframe(df.head())
else:
    st.info("CSV 파일을 업로드하면 미리보기가 표시됩니다.")
if uploaded is not None: 가드 없으면 파일 없을 때 오류 발생.

5. Streamlit Community Cloud 배포

Step 1 — requirements.txt 작성

프로젝트 루트(app.py와 같은 폴더)에 requirements.txt 파일 생성:

streamlit
pandas
plotly

Step 2 — GitHub push

git add requirements.txt
git commit -m "add requirements.txt for deployment"
git push

Step 3 — 배포

  1. share.streamlit.io → GitHub 로그인
  2. New app → 저장소 선택 → Branch: main → Main file path: app.py
  3. Deploy! → 빌드 로그 확인 (1~3분) → URL 생성
  4. URL 복사 → LMS 제출

흔한 배포 오류

오류 원인 해결
ModuleNotFoundError requirements.txt 누락 또는 오타 파일 확인 후 re-push
FileNotFoundError: data/xxx.csv data/ 폴더가 GitHub에 없음 git add data/ → push
앱이 느림 @st.cache_data 없음 데이터 로딩 함수에 추가

BASIC marketing 선택 시

Step 1 — 데이터 로드 + 전처리

import streamlit as st
import pandas as pd

@st.cache_data
def load_marketing():
    df = pd.read_csv('data/marketing_campaign_dataset.csv')
    df['Acquisition_Cost'] = (
        df['Acquisition_Cost']
        .str.replace('[$,]', '', regex=True)
        .astype(float)
    )
    df['Date'] = pd.to_datetime(df['Date'])
    return df

df = load_marketing().copy()
st.title("📣 마케팅 캠페인 대시보드")
st.write(f"전체 데이터: {len(df):,}행")

Step 2 — 사이드바 필터

with st.sidebar:
    st.header("필터")
    if st.button("필터 초기화"):
        st.session_state['campaign_types'] = df['Campaign_Type'].unique().tolist()
        st.session_state['location'] = "전체"

    campaign_types = st.multiselect(
        "캠페인 유형",
        df['Campaign_Type'].unique().tolist(),
        default=st.session_state.get('campaign_types', df['Campaign_Type'].unique().tolist()),
        key='campaign_types'
    )
    location = st.selectbox(
        "지역",
        ["전체"] + sorted(df['Location'].unique().tolist()),
        key='location'
    )

filtered = df[df['Campaign_Type'].isin(campaign_types)]
if location != "전체":
    filtered = filtered[filtered['Location'] == location]

Step 3 — metric

col1, col2, col3 = st.columns(3)
col1.metric("총 캠페인 수", f"{len(filtered):,}")
col2.metric("평균 ROI", f"{filtered['ROI'].mean():.2f}")
col3.metric("평균 전환율", f"{filtered['Conversion_Rate'].mean():.1%}")

Step 4 — 차트

import plotly.express as px

fig = px.bar(
    filtered.groupby('Campaign_Type')['ROI'].mean().reset_index(),
    x='Campaign_Type', y='ROI',
    title='캠페인 유형별 평균 ROI'
)
st.plotly_chart(fig)

BASIC ecommerce 선택 시

Step 1 — 데이터 로드 + 전처리

@st.cache_data
def load_ecommerce():
    df = pd.read_csv('data/ecommerce_sales_data.csv', index_col=0)
    df['City'] = df['City'].str.strip()
    return df

df = load_ecommerce().copy()
st.title("🛒 이커머스 매출 대시보드")

Step 2 — 사이드바 필터

with st.sidebar:
    st.header("필터")
    if st.button("필터 초기화"):
        st.session_state['categories'] = df['Product Category'].unique().tolist()
        st.session_state['month_range'] = (1, 12)

    categories = st.multiselect(
        "카테고리",
        df['Product Category'].unique().tolist(),
        default=st.session_state.get('categories', df['Product Category'].unique().tolist()),
        key='categories'
    )
    month_range = st.slider("월 범위", 1, 12, (1, 12), key='month_range')

filtered = df[
    df['Product Category'].isin(categories) &
    df['Month'].between(month_range[0], month_range[1])
]

Step 3 — metric + 차트

col1, col2 = st.columns(2)
col1.metric("총 주문 수", f"{len(filtered):,}")
col2.metric("총 매출", f"${filtered['Sales'].sum():,.0f}")

import plotly.express as px
fig = px.bar(
    filtered.groupby('Product Category')['Sales'].sum().reset_index(),
    x='Product Category', y='Sales',
    title='카테고리별 총 매출'
)
st.plotly_chart(fig)

막혔을 때 체크리스트

  • KeyError: 'xxx' → 초기화 가드 또는 .get('xxx', 기본값) 추가했는가?
  • 버튼 눌러도 반응 없음 → if st.button(): 블록 안에서 session_state 수정했는가?
  • 멀티페이지 FileNotFoundError → app.py와 같은 폴더? pg.run() 호출했는가?
  • 배포 후 ModuleNotFoundError → requirements.txt에 패키지 있는가? git push 했는가?
  • 배포 후 FileNotFoundError → git add data/ 후 push 했는가?
  • marketing 숫자 이상 → str.replace('[$,]', '', regex=True).astype(float) 전처리 했는가?
  • ecommerce City 이상 → df['City'].str.strip() 적용했는가?
  • uploaded None 오류 → if uploaded is not None: 가드 있는가?

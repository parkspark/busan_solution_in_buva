#  TIL: Streamlit 기초 및 인터랙티브 대시보드 구축

## 요약 (Summary)
Python 코드만으로 HTML, CSS, JS 없이 빠르게 웹 애플리케이션과 대시보드를 구축할 수 있는 **Streamlit**의 핵심 기능을 학습.

다중 데이터셋(IMDB, Titanic)을 활용하여 조건부 필터링, 동적 레이아웃, Plotly 시각화가 결합된 인터랙티브 대시보드를 완성.

---

## 💡 주요 학습 내용 (Key Learnings)

### 1. Streamlit의 실행 모델 (Top-to-Bottom Rerun)
* 사용자가 사이드바의 위젯(슬라이더, 체크박스 등) 값을 변경할 때마다 **`app.py` 스크립트 전체가 위에서 아래로 다시 실행(Rerun)**됨.
* 이 단순한 구조 덕분에 복잡한 상태 관리(State Management) 없이도 코드 흐름에 따라 직관적인 UI 업데이트가 가능.

### 2. 데이터 로딩과 캐싱 (`@st.cache_data`)
* **캐시의 필요성:** 매번 Rerun이 발생할 때마다 대용량 CSV 파일을 다시 읽어오면 병목 현상이 발생.
* **해결:** 로딩 함수 위에 `@st.cache_data`를 선언하여 첫 실행 시 데이터를 메모리에 저장하고, 이후에는 저장된 데이터를 즉시 반환하게 함.
* **중요한 팁 (`.copy()`) :** 캐시된 원본 데이터가 오염되는 것(Mutation)을 막기 위해, 불러온 데이터는 반드시 `load_imdb().copy()` 처럼 복사본을 만들어 사용해야 안전합니다.
* 하나의 앱 안에서 여러 개의 독립적인 데이터(IMDB, apt, titanic 등)를 로드해도 함수명과 변수명만 다르면 서로 간섭 없이 캐싱이 유지됩니다.

### 3. 직관적인 UI 및 레이아웃 구성
* **사이드바 (`with st.sidebar:`):** 메인 화면을 깔끔하게 유지하기 위해 사용자 입력 위젯들을 좌측 패널에 모아 배치.
* **다단 구성 (`st.columns`):** 화면을 가로로 분할하여 KPI 지표(`st.metric`)를 한눈에 나란히 배치할 수 있다.
* **탭 분리 (`st.tabs`):** 한 화면에 스크롤이 너무 길어지는 것을 방지하기 위해 차트 영역과 데이터프레임 확인 영역을 탭으로 분리하여 가독성을 높임.

### 4. 다양한 인터랙티브 위젯 연동
* `st.slider`: 숫자 범위 선택 (예: 나이 범위, 개봉 연도)
* `st.multiselect`: 다중 선택 (예: 객실 등급 리스트 반환)
* `st.selectbox`: 단일 선택 (예: 성별)
* 위젯의 반환값을 Pandas의 데이터프레임 조건식에 바로 연결하여, 유저가 조작하는 즉시 데이터가 필터링되도록 구성.

### 5. Plotly 데이터 시각화 연동
* Plotly Express로 만든 인터랙티브 차트(Scatter, Histogram, Pie)를 웹 앱에 띄우는 과정이 매우 간단.
* `st.plotly_chart(fig, width='stretch')` 코드를 통해 생성된 차트가 브라우저 너비에 맞게 동적으로 렌더링됨.

## ⚠️ 트러블슈팅Troubleshooting)

* **실행 에러 (VSCode 등에서 직접 `.py` 실행 시)**
  * **증상:** `Thread 'MainThread': missing ScriptRunContext!` 경고 발생.
  * **원인:** Streamlit 앱은 일반적인 파이썬 환경이 아닌 자체 백엔드 서버 위에서 구동되어야 하기 때문.
  * **해결:** 에디터의 '실행' 버튼을 누르지 말고, 반드시 터미널(CMD)에서 `streamlit run 파일명.py` (또는 `uv run streamlit run 파일명.py`) 명령어를 입력하여 브라우저를 띄워야 함.
* **들여쓰기(Indentation) 주의**
  * `with` 블록 안에서 작성해야 할 코드(사이드바 내용, 탭 안의 차트 등)가 밖으로 빠져나오지 않도록 파이썬 들여쓰기를 철저히 확인해야 의도한 레이아웃이 깨지지 않음.
